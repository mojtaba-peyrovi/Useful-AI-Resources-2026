Some notes on image generation:

### Key Parameters for Better Results

num_inference_steps: Typically set to 30–50. Higher values improve quality but take longer to generate. <br>

guidance_scale: Usually 7.5. Higher values make the image follow your prompt more strictly. <br>

negative_prompt: Use this to tell the model what not to include (e.g., negative_prompt="blurry, distorted hands, low quality"). 


Example:

```

from diffusers import DiffusionPipeline

# Load the higher quality SDXL model
pipe_xl = DiffusionPipeline.from_pretrained(
    "stabilityai/stable-diffusion-xl-base-1.0", 
    torch_dtype=torch.float16, 
    use_safetensors=True, 
    variant="fp16"
)
pipe_xl.to("cuda")



# 1. Define your prompts and parameters
prompt = "a majestic cat wearing a space suit on Mars, highly detailed, 8k"

# Tell the model what NOT to include
negative_prompt = "blurry, low quality, distorted, extra limbs, ugly, text, watermark"

# 2. Run the pipeline with parameters
result = pipe(
    prompt=prompt,
    negative_prompt=negative_prompt, # Excludes unwanted elements
    num_inference_steps=50,          # Quality (higher = more detail, slower)
    guidance_scale=7.5               # Prompt adherence (7-9 is the "sweet spot")
).images[0]

# 3. Save and display
result.save("/content/drive/MyDrive/improved_space_cat.png")
display(result)
```


## What is a multi-modal model?
a multi-modal model is an AI system that can understand or generate information across different "modes" of data—such as text, images, and audio—within a single framework.

- Unlike "unimodal" models (like ChatGPT originally was for just text), a multi-modal model can "see" an image you upload and discuss it with you, or "hear" a voice command and respond. 


-  The most popular open-source multi-modal models in 2026 are __LLaVA (Large Language-and-Vision Assistant) and BLIP__. You can run them using the Hugging Face transformers library. This Multi-modal model acts like a chatbot that has "eyes"—it can reason about the contents, read text within the image, and describe complex relationships between objects. 


## What is Modetation API?
A Moderation API is a specialized tool used to automatically identify and filter harmful content—such as hate speech, violence, or sexual material—within text, images, or video.


__Why use it?__ If you are building a public application (like a chatbot or social feed) in 2026, using a moderation API is the industry standard for preventing your AI from being "jailbroken" or generating harmful content.

Here is a sample code:
```
import requests

api_url = "api.openai.com"
headers = {
    "Content-Type": "application/json",
    "Authorization": "Bearer YOUR_API_KEY"
}

# You can send text, image URLs, or both
data = {
    "model": "omni-moderation-latest",
    "input": [
        {"type": "text", "text": "Your message here"},
        # {"type": "image_url", "image_url": {"url": "https://example.com/image.jpg"}}
    ]
}

response = requests.post(api_url, headers=headers, json=data)
print(response.json())

```
