### Fine tuning the LLM:

In 2026, it helps to think of LLM controls as two overlapping layers:

__Sampling controls (classic):__ tune how text is sampled from the model's distribution (for example, temperature and top-p).


__Reasoning / verbosity controls (newer flagships):__ tune how much the model is allowed to "think" before answering (a thinking budget / reasoning effort) and how expansive the final answer should be (verbosity). Many providers still support sampling controls too, but for difficult tasks these "thinking" controls can matter more than temperature.


### Temperature:
Temperature affects the randomness of the model's output. Conceptually, it reshapes the probability distribution used for choosing the next token. Lower values (for example, 0.2) produce more consistent, deterministic responses, while higher values (for example, 0.8) encourage more diverse and creative outputs.

For factual applications like summarizing articles or providing technical explanations, lower temperatures usually help keep responses precise and repeatable.


Conversely, for storytelling or brainstorming, higher temperatures can help the model explore less predictable ideas.

__Note.__ Some models are fine-tuned to work best at specific temperature ranges. Always check the provider's documentation for recommended settings. For example, Gemini 3 Pro still exposes temperature, but Google recommends keeping it at the default 1.0 because the model's reasoning is optimized around that setting.

### Top-p (nucleus sampling): controlling probable token selection:
Top-p sampling limits the token pool based on cumulative probability.

For instance, a top-p value of 0.9 means the model considers only the smallest set of tokens whose combined probability mass reaches 90%. This complements temperature by narrowing the choice set before sampling. A high top-p retains diversity, while a lower top-p restricts output to the most likely continuations, which can be useful for precision-critical tasks.

 For deterministic tasks like coding or strict factual answering, use a lower temperature and keep top-p moderately high, or leave temperature at default and reduce top-p slightly.

 ### Max tokens: managing response length:
The max tokens setting defines the maximum length of the model's output in tokens.

### Frequency penalty:
It discourages the model from repeating the same words and phrases too often. Increasing it can reduce looping and repetitive phrasing in long outputs.

### Presence penalty:

encourages the model to introduce new concepts instead of staying on the same ideas. Increasing it can help brainstorming and ideation feel less stuck.

If a model is repeating itself, a small positive frequency penalty is often enough. If the model keeps circling the same points, a small positive presence penalty can help it explore adjacent ideas.


Many state-of-the-art models now expose a control that limits how much the model can "think" before it starts answering. You can think of this as a thinking budget (how many tokens it can spend internally) or a reasoning effort setting (how hard it should try).

### A practical workflow is:

Set reasonable length limits and a structure output schema (if needed).
If you're using a model with reasoning parameters and reasoning "quality" is the problem, increase reasoning effort / thinking budget.
Only then tune sampling (temperature or top-p) if you need more or less diversity in phrasing.


### Verbosity: controlling how detailed the final answer is:
Some newer systems expose a verbosity control that influences how short or detailed the final answer should be, without changing the underlying reasoning strength.
